# Spec Kit Command Prompts

Use these prompts if working through Spec Kit slash commands.

## Constitution

```txt
/speckit.constitution Create governing principles for SonicField Studio: a browser-based audio-driven raster and vector pattern generator. Require one shared simulation engine, separate raster and vector renderers, native SVG export from geometry, no raster tracing, no backend/auth/payments for MVP, strict TypeScript, performance guards, export integrity, and designer-first UX.
```

## Specify

```txt
/speckit.specify Build SonicField Studio, a web app that generates beautiful sound-wave-inspired visual patterns. Users can change amplitude, frequency, phase, speed, symmetry, density, particle size, and noise. The app supports mathematical wave mode, oscillator mode, uploaded audio, and microphone input. It must export both raster PNG and native vector SVG. SVG must be generated from geometry, not from tracing a raster screenshot. The MVP must include pattern modes: Wave Grid, Radial Cymatics, Lissajous, Sphere Field, and Noise Flow. Include presets, randomize, reset, and JSON preset export/import. No login, no backend, no payments.
```

## Plan

```txt
/speckit.plan Use Next.js, React, TypeScript, Tailwind CSS, Zustand, Web Audio API, Canvas 2D starter renderer with a future Three.js/WebGL path, and native SVG export using custom geometry generation with optional Two.js/Paper.js integration. Architecture must separate UI, state, audio engine, simulation engine, raster renderer, vector exporter, presets, and export settings. The simulation engine is the source of truth for both raster and vector outputs.
```
